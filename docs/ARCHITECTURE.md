## Плагинная система

Каждый плагин реализует интерфейс `DevicePlugin`:

```python
class DevicePlugin(ABC):
    async def discover() -> list[dict]      # Поиск устройств
    async def get_state(device_id, config)  # Чтение состояния
    async def set_state(device_id, config, state)  # Управление
```

Добавление нового производителя = один файл в `backend/app/plugins/`.

## Сетевая топология: два роутера

### Проблема

- Роутер 1: ваш ПК, телефон, рабочие устройства
- Роутер 2: IoT-устройства (лампы, розетки, датчики)
- Сети изолированы друг от друга

### Решения

#### 1. Сервер в IoT-сети (рекомендуется)

Разверните DomoHub на Raspberry Pi / мини-ПК / NAS, подключённом к роутеру 2.
Доступ к UI — через Tailscale, ZeroTier или проброс портов.

```
ПК (роутер 1) ──Tailscale──► RPi (роутер 2) ──► IoT устройства
```

#### 2. MQTT как мост

MQTT-брокер на роутере 2. DomoHub на VPS подключается к брокеру.
Все MQTT-устройства (Zigbee2MQTT, Tasmota) доступны удалённо.

#### 3. Облачные API

Tuya Cloud API, Xiaomi Cloud — устройства доступны через интернет
независимо от локальной сети.

#### 4. Dual-homed сервер

Машина с двумя сетевыми интерфейсами (или VLAN) — видит обе сети.

## Потоки данных

### Управление устройством (UI → устройство)

1. Пользователь нажимает кнопку в UI
2. Frontend → `PATCH /api/v1/devices/{id}/state`
3. DeviceService находит плагин устройства
4. Плагин отправляет команду (HTTP/MQTT/Tuya LAN)
5. Состояние сохраняется в БД
6. WebSocket broadcast всем клиентам

### n8n / LLM управление

1. n8n workflow получает голосовой/текстовый ввод
2. LLM определяет intent (включить свет в гостиной)
3. n8n → `POST /api/v1/n8n/command`
4. Backend находит устройство по имени/комнате
5. Выполняет действие через плагин
